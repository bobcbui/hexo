java io 演进

2020-04-29

  1. JDK1.0—JDK1.3, java io类库非常原始，很多UNIX网络编程概念或接口在io库都没有体现，如：Buffer，Channel，Selector等

  1. JDK1.4 NIO 以JSR-51 的身份正式发布，添加了java.nio包

    1. 进行异步IO的缓冲区ByteBuffer

      缓存区容量(capacity)

      缓冲区位置(position)

      缓冲区限制(limit)

      堆外缓存及堆内缓存

    2. 进行异步IO的管道Pipe

    3. 进行各种IO操作的Channel，包括ServerSocketChannel和SocketChannel

      1. Channel 通道，提供了NIO的非阻塞方法API，如文件FileChannel，SocketChannel，涵盖了UDP/TCP网络和文件IO，使用缓冲区进行IO操作

    4. 多种字符集的编码和解码

    5. 实现非阻塞IO操作的多路复用选择器Selector

      Selector选择器，可以通过Channel注册的方式进行检查管道的状态，从而实现单个线程管理多个Channel，从而提高NIO利用效率

  1. 基于流行的Perl实现的正则表达式

  2. 文件通道FileChannel

  3. 。。。。。

  1. 新的NIO库极大的促进了java异步非阻塞编程的能力，但是还是有不完善的地方，特别是对文件的处理能力不足

    1. 没有统一的文件属性（如读写权限）
    2. API能力较弱，如目录的级联创建，递归遍历
    3. 底层存储系统的一些高级API无法使用
    4. 所有文件操作都是同步非阻塞，不支持异步文件读取

  2. 2011年JDK1.7发布。将原来的NIO类进行升级，被称为NIO2.0由JSR-203演进而来

    1. 提供可以批量获取文件属性的API，提供了标准文件系统SPI，供各个服务提供商扩展
    2. 提供AIO功能，支持基于文件的异步IO操作和针对网络套接字Socket的异步操作
    3. 完成JSR-51定义的通道功能，包括对配置和多播数据报的支持
    4. 提供异步套接字通道实现
      1. 通过juc.Future类实现异步操作的结果
      2. 在执行异步操作的时候传入CompletionHandler接口的实现类作为操作完成的回调

AIO实现demo

不需要通过多路复用选择器Selector对注册通道进行轮询操作即可实现异步读写，简化了NIO的编程模型
